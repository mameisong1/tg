# 商品选项功能测试用例

## 测试概述

**测试目标**：验证H5下单页面的商品选项功能（温度、糖度）正常工作

**测试环境**：开发环境 http://localhost:8089

**数据库**：/TG/tgservice/db/tgservice.db

**测试日期**：2026-04-12

---

## 数据结构说明

### product_options 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| category | TEXT | 分类名称（奶茶店、饮料） |
| product_name | TEXT | 商品名称 |
| temperature | TEXT | 温度选项（如：冰/去冰/常温/热） |
| sugar | TEXT | 糖度选项（如：无糖/少糖/正常糖） |

### carts 表 options 字段
- 类型：TEXT（JSON格式）
- 存储：`{"temperature": "冰", "sugar": "少糖"}`

### orders 表 items 字段
- 类型：TEXT（JSON数组）
- 格式：`[{"name": "珍珠奶茶", "quantity": 1, "price": 15, "options": {"temperature": "冰", "sugar": "少糖"}}]`

## 测试用例列表

### TC-01：没有选项的商品可以正常下单

**优先级**：P0

**前置条件**：
1. 开发环境服务正常运行（http://localhost:8089）
2. 数据库中有"酒水"或"零食"分类的商品（无选项配置）
3. 用户已扫码绑定台桌（或员工已选择台桌）

**测试步骤**：
1. 打开 H5 商品点单页面 `/pages/products/products`
2. 点击"酒水"或"零食"分类，选择一个商品（如：啤酒、可乐）
3. 点击商品卡片的 "+" 按钮，添加到购物车
4. 查看购物车浮动按钮，确认数量显示正确
5. 点击购物车图标，进入 `/pages/cart/cart`
6. 确认商品名称、价格、数量显示正确
7. 点击"下单"按钮
8. 等待下单成功提示

**预期结果**：
1. 商品成功添加到购物车
2. 购物车显示商品名称、价格、数量
3. 下单成功，显示"下单成功！请等待服务员送餐"
4. 数据库 orders 表新增一条记录，items 字段包含该商品（无 options）

### TC-02：奶茶店商品可以选择选项后下单

**优先级**：P0

**前置条件**：
1. 开发环境服务正常运行
2. 数据库 product_options 表有奶茶店商品的选项配置
3. 用户已扫码绑定台桌

**测试步骤**：
1. 打开 H5 商品点单页面
2. 点击"奶茶店"分类
3. 选择"珍珠奶茶"商品
4. **弹出选项选择器**：
   - 温度选择："冰"（从选项：多冰/正常冰/少冰/热）
   - 糖度选择："少糖"（从选项：少糖/正常糖/无）
5. 确认选项后点击"确定"按钮
6. 商品添加到购物车，购物车显示："珍珠奶茶（冰，少糖）"
7. 进入购物车页面
8. 确认商品显示为："珍珠奶茶 | 冰 · 少糖 | ¥XX"
9. 点击下单

**预期结果**：
1. 选项选择器正常弹出，显示温度和糖度选项
2. 选择选项后，购物车显示商品名+选项
3. 下单成功
4. 数据库 orders 表 items 字段包含选项：`{"name": "珍珠奶茶", "quantity": 1, "price": XX, "options": {"temperature": "冰", "sugar": "少糖"}}`

### TC-03：带有选项的商品可以从购物车删除、增减数量、正常下单

**优先级**：P0

**前置条件**：
1. 购物车中已有带选项的奶茶店商品
2. 用户已绑定台桌

**测试步骤**：

**步骤3.1：增加数量**
1. 进入购物车页面
2. 找到带选项的商品（如：生椰拿铁）
3. 点击 "+" 按钮，数量从1变为2
4. 确认总价正确更新

**步骤3.2：减少数量**
1. 点击 "-" 按钮，数量从2变为1
2. 确认总价正确更新

**步骤3.3：删除商品**
1. 点击"删除"按钮
2. 确认商品从购物车移除
3. 确认总价更新为0（如果只有一个商品）

**步骤3.4：重新添加并下单**
1. 返回商品页面
2. 重新添加带选项的商品
3. 进入购物车，确认选项显示正确
4. 点击下单

**预期结果**：
1. 增减数量功能正常，价格计算正确
2. 删除功能正常，商品移除后总价更新
3. 下单成功，数据库正确存储选项信息

### TC-04：饮料分类所有商品都可以获取冷热选项并下单

1. product_options 表有饮料分类的通用配置（temperature="冰镇/常温", sugar="-"）

**测试步骤**：

2. 点击"饮料"分类
3. **遍历所有饮料商品**：

   - 检查是否弹出选项选择器
   - 检查温度选项是否为"冰镇/常温"
   - 选择"冰镇"或"常温"
   - 确认添加到购物车
4. 验证至少3个不同饮料商品的选项功能

**预期结果**：
1. 所有饮料商品都能获取到温度选项（冰镇/常温）
2. 糖度选项显示为"-"或隐藏
3. 选择温度后成功添加到购物车
4. 购物车显示："{商品名}（冰镇）"或"{商品名}（常温）"

## 附录：API 接口

### 获取商品选项
```
GET /api/product-options?category=奶茶店
GET /api/product-options?productName=珍珠奶茶
```

### 添加购物车（带选项）
```json
POST /api/cart
{
  "sessionId": "sess_xxx",
  "tableNo": "普台1",
  "productName": "珍珠奶茶",
  "quantity": 1,
  "options": {
    "temperature": "冰",
    "sugar": "少糖"
  }
}
```

### 获取购物车
```
GET /api/cart/:sessionId
```
返回：
```json
{
  "items": [
    {
      "product_name": "珍珠奶茶",
      "quantity": 1,
      "price": 15,
      "options": "{\"temperature\":\"冰\",\"sugar\":\"少糖\"}",
      "image_url": "..."
    }
  ],
  "totalPrice": 15
}
```

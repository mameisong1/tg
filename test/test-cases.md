# 商品选项功能测试用例

## 测试用例列表

### TC-01: 没有选项的商品可以正常下单
**优先级**: P0

**前置条件**:
- 购物车为空
- 选择"零食"或"其他"分类中没有选项的商品

**测试步骤**:
1. 浏览器打开 http://localhost:8089/pages/products/products
2. 选择一个没有选项的商品（如薯片）
3. 点击"加入购物车"
4. 验证：不弹出选项窗口，直接提示"已加入购物车"
5. 进入购物车页面
6. 验证：购物车显示该商品，无选项文字
7. 点击"提交订单"
8. 验证：订单创建成功

**预期结果**: 
- 不弹出选项弹窗
- 购物车正常显示
- 订单正常创建

**截图**: `tc01-no-options-order.png`

### TC-02: 奶茶店商品可以选择选项后下单
**优先级**: P0

**前置条件**:
- 购物车为空
- 奶茶店分类下有带温度和糖度选项的商品

**测试步骤**:
1. 浏览器打开 http://localhost:8089/pages/products/products
2. 选择"奶茶店"分类
3. 选择一个带选项的商品（如珍珠奶茶）
4. 点击"加入购物车"
5. 验证：弹出底部选项弹窗，显示温度和糖度选项
6. 验证：默认选中第一个温度和第一个糖度
7. 修改温度和糖度选择
8. 点击"确定"
9. 验证：提示"已加入购物车"
10. 进入购物车页面
11. 验证：购物车显示商品名和选项（如"珍珠奶茶 正常冰+少糖"）
12. 点击"提交订单"
13. 验证：订单创建成功

**预期结果**:
- 弹出选项弹窗
- 选项可正常选择
- 购物车显示选项
- 订单正常创建

**截图**: `tc02-milk-tea-options.png`

### TC-03: 带有选项的商品可以从购物车正常删除、增减数量，可以正常下单
**优先级**: P0

**前置条件**:
- 购物车中已有带选项的商品

**测试步骤**:
1. 浏览器打开 http://localhost:8089/pages/cart/cart
2. 找到带选项的商品
3. 点击"+"增加数量
4. 验证：数量正确+1
5. 点击"-"减少数量
6. 验证：数量正确-1，减到0时自动删除
7. 点击"删除"按钮
8. 验证：该商品从购物车消失
9. 验证：其他商品不受影响

**预期结果**:
- 增减数量正确
- 删除操作正确
- 不影响其他商品

**截图**: `tc03-cart-manage.png`

### TC-04: 饮料分类里的所有商品都可以获取到冷热选项，可以选择温度下单
**优先级**: P0

**前置条件**:
- 数据库中存在 `category='饮料' AND product_name='所有商品'` 的通配记录

**测试步骤**:
1. 浏览器打开 http://localhost:8089/pages/products/products
2. 选择"饮料"分类
3. 任选一个饮料商品（任意商品名）
4. 点击"加入购物车"
5. 验证：弹出选项弹窗，显示温度选项
6. 选择温度
7. 点击"确定"
8. 验证：加入购物车成功
9. 换一个不同的饮料商品重复步骤3-8
10. 验证：同样能弹出温度选项

**预期结果**:
- 饮料分类下所有商品都能匹配到冷热选项
- 通配规则生效

**截图**: `tc04-drink-wildcard.png`

### TC-05: 数据库订单表正常存储选项内容
**优先级**: P0

**前置条件**:
- 已创建一个包含选项的订单

**测试步骤**:
1. 浏览器完成一次带选项商品的下单流程
2. 在服务器上执行：
```bash
cd /TG/tgservice && sqlite3 tgservice.db "SELECT items FROM orders ORDER BY id DESC LIMIT 1;"
```
3. 验证：返回的JSON中包含 `"options":"正常冰+少糖"` 或类似内容

**预期结果**:
- 订单items JSON中包含options字段
- options值与用户选择一致

**截图**: `tc05-db-order-options.png`（终端截图）

### TC-06: 收银看板能正常显示商品名+选项内容
**优先级**: P0

**前置条件**:
- 存在一个包含选项的新订单
- 收银看板页面已打开

**测试步骤**:
1. 浏览器打开 http://localhost:8088/admin/cashier-dashboard.html
2. 登录（tgadmin / mms633268）
3. 找到最新订单
4. 验证：商品名后显示选项内容，如"珍珠奶茶（正常冰+少糖）"
5. 验证：没有选项的商品只显示商品名，无多余括号

**预期结果**:
- 有选项的商品显示为 "商品名（选项）"
- 无选项的商品显示为 "商品名"

**截图**: `tc06-cashier-dashboard.png`

## 二、发现的BUG

### BUG-001: 日期范围查询导致当天订单丢失 (严重)

**问题描述**:
`getDateRange()` 函数返回的 `dateEnd` 格式为 `YYYY-MM-DD`，但 SQL 查询使用 `created_at <= dateEnd` 进行字符串比较。

由于 `created_at` 存储格式为 `YYYY-MM-DD HH:MM:SS`，当比较 `created_at <= '2026-05-03'` 时：
- `'2026-05-03 10:00:00' > '2026-05-03'`（字符串比较）
- 导致当天所有订单都不会被查询到

**影响范围**:
- `/api/tea-fruit/my-stats` - 助教当天奶茶/果盘订单统计为0
- `/api/tea-fruit/admin-stats` - 所有助教当天统计为0
- `/api/tea-fruit/coach-detail` - 助教明细当天数据为空

**修复建议**:
```javascript
// 方案1: dateEnd 添加时间部分
function getDateRange(period) {
  // ...
  if (period === 'this-month') {
    return {
      dateStart: dateStr(year, month, 1),
      dateEnd: dateStr(year, month, day) + ' 23:59:59',  // 修改这里
      label: '本月'
    };
  }
  // ...
}

// 方案2: SQL使用DATE函数
WHERE DATE(o.created_at) >= ? AND DATE(o.created_at) <= ?
```

# QA5 测试用例：后台admin助教管理-水牌联动

## 功能描述
修复助教管理页面删除/修改助教后水牌未同步的问题：
1. 删除助教时，删除水牌
2. 助教改为离职时，删除水牌
3. 离职改为全职/兼职时，创建水牌
4. 修改班次时，映射水牌状态

## 后端接口说明
| 接口 | 方法 | 说明 |
|------|------|------|
| `GET /api/admin/coaches` | GET | 获取助教列表 |
| `POST /api/admin/coaches` | POST | 添加助教（自动创建水牌） |
| `PUT /api/admin/coaches/:coachNo` | PUT | 更新助教（联动水牌） |
| `DELETE /api/admin/coaches/:coachNo` | DELETE | 删除助教（仅可删除离职，联动删除水牌） |
| `PUT /api/admin/coaches/:coachNo/shift` | PUT | 修改班次（联动水牌状态映射） |
| `GET /api/water-boards` | GET | 获取水牌列表 |

## 水牌状态映射规则（班次修改时）
| 旧班次水牌状态 | 新班次水牌状态 |
|---------------|---------------|
| 早班空闲 | 晚班空闲 |
| 晚班空闲 | 早班空闲 |
| 早班上桌 | 晚班上桌 |
| 晚班上桌 | 早班上桌 |
| 早加班 | 晚加班 |
| 晚加班 | 早加班 |

## 测试用例

### TC-01：删除离职助教 - 验证coaches和water_boards都删除
- **前提**：存在一名状态为「离职」的助教A
- **步骤**：
  1. 登录后台管理页面
  2. 进入助教管理，筛选「离职」状态
  3. 确认助教A存在，且water_boards中有对应记录
  4. 点击助教A的「删除」按钮
  5. 确认删除
- **预期结果**：
  - 删除成功提示
  - 助教A从coaches表中消失
  - 助教A从water_boards表中消失
- **优先级**：P0

### TC-02：全职改为离职 - 验证水牌删除
- **前提**：存在一名状态为「全职」的助教B
- **步骤**：
  1. 登录后台管理页面
  2. 进入助教管理
  3. 点击助教B的「编辑」按钮
  4. 将状态从「全职」改为「离职」
  5. 保存
- **预期结果**：
  - 更新成功提示
  - 助教B状态变为「离职」
  - water_boards中对应记录被删除
- **优先级**：P0

### TC-07：添加新助教 - 验证水牌自动创建
- **前提**：无
- **步骤**：
  1. 登录后台管理页面
  2. 进入助教管理
  3. 点击「+ 添加助教」
  4. 填写必填字段（工号、艺名），选择状态为「全职」
  5. 保存
- **预期结果**：
  - 添加成功提示
  - 新助教出现在列表中
  - water_boards中新增对应记录
- **优先级**：P1

### TC-08：删除非离职助教 - 验证不允许删除
- **前提**：存在一名状态为「全职」的助教G
- **步骤**：
  1. 登录后台管理页面
  2. 进入助教管理
  3. 确认助教G没有「删除」按钮（因为只有离职助教才有删除按钮）
- **预期结果**：
  - 全职/兼职助教不显示删除按钮
- **优先级**：P1

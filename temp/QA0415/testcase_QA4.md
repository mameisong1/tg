## 已知问题（代码审查发现）

### BUG-01: CSS 媒体查询未编译到 H5 构建产物中

**严重程度**：🔴 高  
**描述**：源文件 `water-board.vue` 和 `water-board-view.vue` 中定义了 `@media (max-width: 420px)` 和 `@media (max-width: 360px)` 响应式规则，但编译后的 CSS 文件 `water-board-DjU98EkM.css` 和 `water-board-view-Cr5Zr8RN.css` 中不包含任何 `@media` 规则。

**影响**：窄屏和极窄屏的 CSS 响应式适配在 H5 端完全不生效。

**复现**：
1. 查看 `/TG/tgservice/frontend/assets/water-board-DjU98EkM.css`
2. 搜索 `@media` → 结果为 0

### BUG-02: 30秒自动刷新定时器未编译到 H5 构建产物中

**严重程度**：🔴 高  
**描述**：源文件中 `onMounted` 共享初始化块包含 `setInterval(loadData, 30000)` 用于30秒自动刷新，但编译后的 JS 文件中不存在 `setInterval`、`clearInterval` 或 `30000`。

**影响**：H5 端水牌管理/查看页面不会自动刷新数据。

**复现**：
1. 查看 `/TG/tgservice/frontend/assets/pages-internal-water-board.ciioox3r.js`
2. 搜索 `setInterval` 或 `30000` → 结果为 0

---

## 测试结论

由于 BUG-01 和 BUG-02 存在于当前 H5 构建产物中，大部分测试用例（TC-02 至 TC-12）在构建产物未修复前将全部失败。

建议：
1. 排查 uniapp H5 构建配置，确认 scoped style 中的 `@media` 是否被正确处理
2. 排查 `onMounted` 多块合并问题，确保共享初始化块被正确编译
3. 修复后重新执行全部测试用例

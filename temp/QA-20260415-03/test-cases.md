# 乐捐报备 — 照片上传（最多3张）测试用例

**需求**：前台H5助教用的乐捐报备页面，在乐捐详情页面（lejuan-proof.vue）将只能提交1张照片，改为最多提交3张照片。必须使用公共照片提交模块 `useImageUpload`（`/TG/tgservice-uniapp/src/utils/image-upload.js`）。

**测试环境**：
- 后端API：`http://127.0.0.1:8088`
- 前端H5：`http://127.0.0.1:8089`
- 短信验证码：`888888`

**涉及页面/模块**：
| 文件 | 说明 |
|------|------|
| `pages/internal/lejuan-proof.vue` | 乐捐详情页 — 提交付款截图 |
| `pages/internal/lejuan.vue` | 乐捐报备主页 — 记录列表 |
| `pages/internal/lejuan-list.vue` | 乐捐一览（管理视角） |
| `utils/image-upload.js` | 公共图片上传 composable（useImageUpload） |
| `backend/routes/lejuan-records.js` | 后端 PUT /lejuan-records/:id/proof |

## TC-01 上传1张照片（正常流程）

| 项目 | 内容 |
|------|------|
| **用例编号** | TC-01 |
| **用例名称** | 上传1张照片并成功提交 |
| **优先级** | P0 — 阻塞 |
| **前置条件** | 1. 助教已登录H5（用户名/密码 + 短信验证码 888888）<br>2. 存在一条状态为 `active` 或 `returned` 且近2天内的乐捐记录（无可通过乐捐报备页面新建）<br>3. 该记录尚未上传付款截图 |
| **测试步骤** | 1. 打开 `http://127.0.0.1:8089`，进入「会员」页<br>2. 点击「乐捐报备」卡片，进入乐捐报备页面<br>3. 在近2天记录列表中找到一条没有截图的记录（显示"📷 点击传截图"），点击进入<br>4. 确认进入「提交付款截图」页面，页面顶部显示记录详情（助教、预约时间等）<br>5. 确认上传区域显示"上传付款截图（最多3张）"标题<br>6. 确认页面底部显示一个虚线边框的"📷 上传图片"按钮<br>7. 点击"上传图片"按钮<br>8. 在弹出的选择器中选择「相册」，选择1张JPG格式照片<br>9. 等待上传完成（观察进度条从0%到100%，toast提示"上传成功"）<br>10. 确认图片网格中出现1张已上传图片缩略图，右上角有"✕"删除按钮<br>11. 确认"上传图片"按钮仍然可见（因未达上限）<br>12. 点击底部「提交截图」按钮 |
| **预期结果** | - 步骤6：页面显示上传标题和1个上传按钮<br>- 步骤9：上传进度条显示0%→100%，toast提示"上传成功"<br>- 步骤10：图片网格中有1张缩略图，右上角有删除按钮<br>- 步骤11："上传图片"按钮仍然显示<br>- 步骤12：弹出"提交成功"弹窗，提交后返回列表页，该记录显示"✅ 已传截图"<br>- 后端 `proof_image_url` 字段存储值为 `["url1"]`（JSON数组含1个元素） |

## ⚠️ 代码审查发现的问题（需在改造时修复）

### 问题1：`watch` 未导入
**文件**：`lejuan-proof.vue`
**行号**：第136行
**描述**：代码使用了 `watch(proofUrls, ...)` 但 `import { ref, onMounted } from 'vue'` 中没有导入 `watch`。
**影响**：已有照片无法回显到 `imageUrls` 数组中，导致重新进入页面时看不到已上传的照片。
**修复建议**：改为 `import { ref, onMounted, watch } from 'vue'`

### 问题2：`lejuan.vue` 中 `goToProof` 未解析JSON
**文件**：`lejuan.vue`
**行号**：第228-230行
**描述**：`uni.previewImage({ urls: [rec.proof_image_url] })` 直接将 `proof_image_url` 作为URL传入。如果 `proof_image_url` 是 JSON 数组字符串 `["url1","url2","url3"]`，previewImage 收到的将是字符串数组而非URL数组。
**修复建议**：先 JSON.parse 提取 URL 数组：
```javascript
let urls = []
try { urls = JSON.parse(rec.proof_image_url) } catch { urls = [rec.proof_image_url] }
uni.previewImage({ urls })
```

### 问题3：`lejuan-list.vue` 中 `proof_image_url` 直接作为 image src
**文件**：`lejuan-list.vue`
**行号**：第103行
**描述**：`<image :src="item.proof_image_url">` 当 `proof_image_url` 是 JSON 数组字符串时，src 不是有效URL，图片无法显示。
**修复建议**：解析 JSON 数组，取第一个 URL 作为缩略图：
```javascript
// computed 或方法
getFirstProofUrl(url) {
  if (!url) return ''
  try {
    const arr = JSON.parse(url)
    return Array.isArray(arr) && arr.length > 0 ? arr[0] : url
  } catch { return url }
}
```

### 问题4：`lejuan-list.vue` 中 `previewProof` 只预览单张
**文件**：`lejuan-list.vue`
**行号**：第168-170行
**描述**：`uni.previewImage({ urls: [url] })` 只传单个URL，无法预览全部照片。
**修复建议**：解析 JSON 数组，传入全部 URL。

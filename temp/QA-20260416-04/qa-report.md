# QA最终报告 - 助教离店后水牌台桌号清除修复

## 1. QA需求概述

**问题**：生产环境有部分助教已离店，水牌状态为「休息、公休、请假、下班」，但水牌里的台桌号还存在，没有清除。

**根因**：当助教状态变为非工作状态时，4条操作路径中有2条遗漏了 `table_no` 的清除逻辑。

## 2. 设计摘要

| 文件 | 修改内容 |
|------|---------|
| `backend/routes/water-boards.js` | off-state（休息/公休/请假/下班）时自动清除 `table_no = NULL`；下班状态额外清除 `clock_in_time = NULL`；如果前端显式传 `table_no` 则尊重输入 |
| `backend/routes/applications.js` | 审批通过公休申请时，SQL增加 `table_no = NULL, clock_in_time = NULL` |

**未修改**：
- `coaches.js` 批量/单个修改班次：只做 working↔working 映射，不涉及 off-state 变更，无需修改

## 一、需求概述

实现天宫员工奖罚管理功能：
1. 系统配置新增奖罚类型设定（JSON 存储）
2. 数据库新增奖罚表
3. 后台用户表新增在职状态
4. 前台 H5 新增奖金设定页面（店长给服务员设日奖）
5. 前台 H5 新增奖金查看页面（服务员/助教查看自己奖罚明细）
6. 后台 admin 新增人事目录 + 奖罚统计页面（人事执行奖罚）

---

## 二、设计摘要

### 数据库变更
- 新建 `reward_penalties` 表（10 字段 + 唯一约束 + 5 个索引）
- `admin_users` 新增 `employment_status` 字段（在职/离职）
- `system_config` 新增 `reward_penalty_types` 配置

### 后端 API（9 个端点）
| 路径 | 方法 | 功能 |
|------|------|------|
| `/api/admin/reward-penalty/types` | GET/PUT | 奖罚类型配置读写 |
| `/api/reward-penalty/upsert` | POST | 写入/更新/删除奖罚记录 |
| `/api/reward-penalty/list` | GET | 查询奖罚记录（按用户过滤） |
| `/api/reward-penalty/stats` | GET | 按月统计 |
| `/api/reward-penalty/batch-execute` | POST | 批量执行 |
| `/api/reward-penalty/execute/:id` | POST | 单条执行 |
| `/api/reward-penalty/unexecute/:id` | POST | 撤销执行 |
| `/api/reward-penalty/targets` | GET | 目标人员列表 |
| `/api/reward-penalty/my-types` | GET | 用户可用奖罚类型 |
| `/api/reward-penalty/batch-set` | POST | 批量设定奖金 |
| `/api/reward-penalty/stats/summary` | GET | 金额汇总 |

### 前端页面
- `reward-penalty-set.vue` — 奖金设定（快捷按钮 10/20/50 + 自定义输入）
- `reward-penalty-view.vue` — 奖金查看（筛选 + 明细 + 统计栏）
- `reward-penalty-stats.html` — 后台奖罚统计（筛选 + 批量执行）

### 编码规范
- ✅ 时间处理全部使用 `TimeUtil.nowDB()`
- ✅ 数据库连接复用 `db/index.js`
- ✅ 写入操作使用 `enqueueRun()`
- ✅ 页面不显示 `coach_no`

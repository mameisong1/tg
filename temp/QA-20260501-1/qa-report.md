# QA-20260501-1 最终报告

## 强制钉钉打卡翻牌

---

## 1. QA需求概述

**核心目标**：用户上班打卡不再提交钉钉打卡截图证明打卡时间，改为系统直接使用钉钉推送的打卡时间或系统主动调用接口获取钉钉打卡时间。

**具体流程**：
1. 已收到钉钉推送 → 直接打卡成功
2. 系统主动获取10分钟内钉钉打卡时间 → 成功获取则打卡成功
3. 都未成功 → 弹框告知用户必须先钉钉打卡
4. 沙漏对话框 + 每10秒轮询 → 5分钟超时则打卡失败

**注意事项**：
- 上班打卡页面显眼提示 + 勾选框
- 截图上传改为非必须
- clock_in_time 统一使用钉钉打卡时间
- 所有场景（上班、乐捐归来、双重场景）都强制钉钉打卡
- 下班打卡维持现状

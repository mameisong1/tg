# 乐捐功能改造 - 测试用例

> **测试环境**：前端 H5 `http://127.0.0.1:8089` | 后端 API `http://127.0.0.1:8088`
> **短信验证码**：888888
> **浏览器要求**：同时不超过2个标签页，用完关闭
> **乐捐状态流转**：待出发（预约）→ 乐捐中 → 已归来
> **乐捐结束方式**：助教点击"上班"按钮

## TC-01: 乐捐报备页面-无预计外出小时数组件

**优先级**：P0 核心

**前置条件**：
- 助教已登录 H5（验证码 888888）
- 当前无任何进行中的乐捐（待出发或乐捐中）

**操作步骤**：
1. 浏览器打开 `http://127.0.0.1:8089`，完成登录
2. 进入乐捐报备页面
3. 检查页面表单中是否存在"预计外出小时数"输入框/选择器
4. 滚动页面，确认表单底部也没有该组件

**预期结果**：
- 页面中**不显示**"预计外出小时数"相关的任何输入组件（包括输入框、下拉选择器、数字步进器等）
- 其他表单组件（如备注、截图上传等）正常显示

## TC-03: 乐捐报备页面-点击上班按钮结束乐捐，回到空闲

**优先级**：P0 核心

**前置条件**：

5. 观察页面跳转或状态变化
6. 检查助教状态是否变为"空闲"
7. 检查班次是否保持不变（仍为"早班"或"晚班"）
8. 检查该乐捐记录状态是否变为"已归来"

**预期结果**：
- 点击"上班"后，助教状态变为"空闲"
- 班次保持不变（早班→早班，晚班→晚班）
- 乐捐记录状态更新为"已归来"

- "乐捐中"状态的乐捐记录排在"待出发"之后、"已归来"之前

- 排序逻辑：待出发 > 乐捐中 > 已归来

## TC-10: 乐捐提交-正常流程（无预计外出小时数）

**优先级**：P0 核心

**前置条件**：
- 助教已登录 H5
- 当前无任何进行中的乐捐（待出发或乐捐中）

**操作步骤**：
1. 浏览器打开 `http://127.0.0.1:8089`，完成登录
2. 进入乐捐报备页面
3. 确认页面无"预计外出小时数"组件
4. 填写必填项（如备注/说明）
5. 上传一张截图（乐捐截图）
6. 点击"提交"按钮
7. 观察提交结果
8. 返回乐捐列表，确认新乐捐记录已创建

**预期结果**：
- 表单中无"预计外出小时数"字段
- 提交成功，无报错
- 乐捐列表中新增一条记录
- 新记录状态为"待出发"（预约）
- 提交的截图在新记录中可见
- 提交后页面有成功提示

**异常流程**：
- 10a. 不填写必填项直接提交 → 应提示必填项不能为空
- 10b. 不上传截图直接提交 → 根据业务规则判断是否允许

## TC-11: 乐捐-上班按钮回到空闲状态（班次保持不变）

- 助教已登录 H5
- 该助教当前班次为"晚班"，状态为"乐捐中"
- 有一个"乐捐中"的乐捐记录

**操作步骤**：
1. 浏览器打开 `http://127.0.0.1:8089`，完成登录
2. 进入乐捐报备页面
3. 记录当前班次为"晚班"，状态为"乐捐中"
4. 点击"上班"按钮
5. 等待操作完成
6. 查看助教当前状态

**预期结果**：
- 助教状态变为"空闲"

- 乐捐记录状态更新为"已归来"

# 天宫国际商品数据同步脚本

## 概述

从台客多后台采集商品数据，同步到天宫国际数据库。

## 文件说明

| 文件 | 说明 |
|------|------|
| `sync-products.js` | 同步脚本 |
| `sync-products.log` | 日志文件 |
| `../data/taikeduo-products.json` | 采集的商品数据 |

## 使用方法

```bash
cd /TG/tgservice
node scripts/sync-products.js
```

## 配置说明

脚本内置配置（位于脚本顶部的 `CONFIG` 对象）：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `chromePort` | 9222 | Chrome 调试端口 |
| `chromeStartCmd` | `bash /root/chrome` | Chrome 启动命令 |
| `targetUrl` | 台客多商品列表页面 | 采集目标页面 |
| `dbPath` | `/TG/tgservice/db/tgservice.db` | 数据库路径 |
| `logFile` | `sync-products.log` | 日志文件路径 |
| `dataFile` | `taikeduo-products.json` | 采集数据保存路径 |
| `minProducts` | 100 | 最少采集数量（少于此值不写入数据库） |
| `pageTimeout` | 60000 | 页面加载超时（毫秒） |
| `excludeCategories` | `['美女教练']` | 排除的商品分类 |

## 执行流程

```
1. 检查/启动 Chrome
2. 打开台客多商品列表页面
3. 翻页采集所有商品数据
   - 商品名称、图片URL、零售价
   - 库存总数、可用库存
   - 分类、创建人、创建时间
4. 校验采集数量（≥100）
5. 过滤排除分类（如"美女教练"）
6. 同步商品表（同名覆盖更新）
7. 同步分类表（过滤排除分类）
8. 输出最终报告
```

## 商品状态规则

| 条件 | 状态 |
|------|------|
| 可用库存 > 0 | 上架 |
| 可用库存 = 0 | 下架 |

## 异常处理

### 1. Chrome 未运行
- 自动执行 `bash /root/chrome` 启动
- 等待最长 30 秒

### 2. 页面加载超时
- 综合使用 load 事件 + 关键组件检测 + 60秒超时

### 3. 采集数量不足
- 采集数量 < 100 时，**只写日志，不写入数据库**
- 脚本以错误码 1 退出

### 4. 需要登录
- 检测到登录页面时抛出异常

## 日志内容

日志文件包含：
- 采集开始/结束时间
- Chrome 启动状态
- 每页采集的商品 JSON 数据
- 异常错误信息
- 分类统计报告
- 最终同步结果

## 定时任务

已配置 crontab 定时执行：

```bash
# 每天 14:00 同步商品信息
0 14 * * * cd /TG/tgservice && /usr/bin/node scripts/sync-products.js >> /TG/tgservice/scripts/sync-products-cron.log 2>&1
```

查看定时任务：
```bash
crontab -l | grep sync-products
```

查看定时执行日志：
```bash
tail -f /TG/tgservice/scripts/sync-products-cron.log
```

## 注意事项

1. **Chrome 必须配置正确**
   - 确保 `/root/chrome` 脚本存在且可执行
   - 端口 9222 未被占用

2. **台客多登录态**
   - 确保 mychrome-profile 中已保存登录状态
   - 如需重新登录，请手动打开 Chrome 登录

3. **数据库备份**
   - 建议定期备份数据库文件
   - 同步操作为覆盖更新，同名商品会被覆盖

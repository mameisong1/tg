## 🔍 根因分析

### nav-control.js 代码分析

文件位置: `/TG/tgservice/admin/nav-control.js`

```javascript
// 角色菜单权限映射
const roleAllowedPages = {
  '管理员': 'all',
  '店长': 'all',
  '助教管理': 'all',
  '前厅管理': ['index.html', 'cashier-dashboard.html', 'products.html', 'orders.html', 'vip-rooms.html', 'tables.html'],
  '收银': ['index.html', 'cashier-dashboard.html', 'products.html', 'orders.html'],
  '教练': ['water-boards.html']
};
```

**问题1: 角色字段名可能不匹配**
- 前端代码使用中文角色名: `'收银'`, `'前厅管理'`, `'教练'`
- 后端JWT token中的role字段可能是英文: `'cashier'`, `'front_admin'`, `'coach'`
- 导致 `roleAllowedPages[role]` 返回 `undefined`，角色被当作管理员处理

**问题2: 收银角色允许页面配置错误**
- 包含 `index.html`（数据概览），但预期收银不应看到数据概览
- 包含 `orders.html`（订单管理），但预期收银不应看到订单管理

**问题3: 前端缓存问题**
- 浏览器可能缓存了旧版本的 `nav-control.js`
- 需要清除缓存或添加强制刷新机制

### 后端登录拦截分析

**生效部分**: 
- 服务员角色被正确拦截（BUG-020已修复）
- 错误提示: "服务员不允许登录后台管理系统，请使用前台系统"

**未生效部分**:
- 登录成功后，前端菜单过滤未执行
- 直接URL访问可能未被后端拦截（需进一步验证）

## 建议

1. **紧急**：检查后端JWT token中的role字段值，确认是否与前端nav-control.js中的角色名匹配
2. **检查**：nav-control.js是否被正确引用到所有admin页面
3. **修复**：收银角色的允许页面不应包含'index.html'（数据概览）
4. **验证**：清除浏览器缓存后重新测试
5. **补充测试**：直接URL访问测试（验证后端API拦截）

**问题2: TC-019和TC-020超出本次修复范围**

- TC-019（收银权限用户仅见收银看板）和TC-020（前厅管理权限菜单验证）验证的是权限控制功能，这属于V2.0需求文档中的**基础功能**，而非本次bug修复的范围。

**问题3: TC-001预期结果中"60秒内至少6次提示音"可能不准确**

- 发声逻辑是在每次轮询（10秒间隔）且有待处理订单时触发。
- 页面加载后立即执行一次 `pollNewOrders`，如果初始有订单则立即发声。
- 之后每10秒轮询一次。
- 60秒内理论上最多7次（立即1次 + 10s/20s/30s/40s/50s各1次 = 7次）。
- 但实际次数受API响应速度影响。
- **建议**: 预期结果改为"60秒内应听到至少5次提示音"，留有API响应延迟的余地。

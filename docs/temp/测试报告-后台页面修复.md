## 三、发现的问题

### BUG-01: 用户管理 — 角色 Badge 样式未区分 ❌

**严重程度**: 中  
**影响页面**: users.html  
**问题描述**: 所有用户角色的 Badge 样式均为 `badge badge-admin`，没有按角色类型使用不同的颜色/样式。

**预期**:
- 超级管理员/管理员 → 红色 Badge
- 店长/助教管理 → 金色 Badge
- 收银 → 蓝色 Badge
- 教练/服务员 → 其他颜色

**实际**: 所有角色统一显示为 `badge badge-admin`

**建议**: 在 users.html 中添加角色到 Badge class 的映射逻辑

---

### BUG-02: 批量更新班次 — 异步加载时序问题 ❌

**严重程度**: 低  
**影响页面**: coaches.html#batch-shift  
**问题描述**: 页面加载时 `checkHash()` 同步调用 `renderBatchShift()`，但 `loadCoaches()` 是异步的，导致数据未加载完成就渲染，显示「暂无助教数据」。

**实际表现**: 首次访问 #batch-shift 页面时显示空数据，刷新后正常

**建议**: 在 `checkHash()` 中确保 `loadCoaches()` 完成后再调用 `renderBatchShift()`

---

### BUG-03: 批量更新班次 — 403 错误 (偶发) ❌

**严重程度**: 中  
**影响页面**: coaches.html#batch-shift  
**问题描述**: 浏览器访问 `/api/admin/coaches` 时偶发 403 Forbidden，但 curl 测试正常。可能是请求头格式或浏览器缓存问题。

**建议**: 检查 API 请求的 Authorization 头格式，清除浏览器缓存后重试

---

### BUG-04: 用户管理 — 角色选择器需确认 ⚠️

**严重程度**: 低  
**问题描述**: 测试用例 TC-036 标注代码现状只有「管理员」和「收银员」两个选项，但实际测试发现添加用户弹窗已包含全部7种角色选项（管理员、店长、助教管理、教练、前厅管理、收银、服务员）。

**结论**: 该问题已修复，代码现状与测试用例描述不一致，需更新测试用例。

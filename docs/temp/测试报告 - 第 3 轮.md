## 关键问题

### 问题 1: 认证中间件与数据库结构不匹配（严重）

**问题描述**: 
`/TG/tgservice/backend/middleware/auth.js` 第 39 行查询 `admin_users` 表时使用了不存在的字段 `name`。

**错误日志**:
```
认证中间件错误：Error: SQLITE_ERROR: no such column: name
--> in Database#get('SELECT username, name, role FROM admin_users WHERE username = ?', [ 'tgadmin' ], ...)
```

**数据库实际结构**:
```sql
CREATE TABLE admin_users (
  username TEXT PRIMARY KEY,
  password TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  role TEXT DEFAULT '管理员'
);
```

**影响范围**: 
- 所有需要认证的 API 都无法使用
- 包括：水牌管理、上下班、服务单、上下桌单、约客管理等所有后台功能
- 第 3 轮测试全部 15 个用例无法执行

**修复建议**:
```javascript
// 修改 /TG/tgservice/backend/middleware/auth.js 第 39 行
// 从:
'SELECT username, name, role FROM admin_users WHERE username = ?',
// 改为:
'SELECT username, role FROM admin_users WHERE username = ?',
```

同时需要修改第 72 行附加用户信息的代码：
```javascript
// 从:
req.user = {
  username: user.username,
  name: user.name,
  role: user.role
};
// 改为:
req.user = {
  username: user.username,
  name: user.username,  // 或使用其他默认值
  role: user.role
};
```

### 问题 2: Token 格式不一致（严重）

**问题描述**:
- `/TG/tgservice/backend/server.js` 中的 `/api/admin/login` 使用 JWT 生成 token
- `/TG/tgservice/backend/middleware/auth.js` 期望的 token 格式是 `username:timestamp:signature`

**代码对比**:
```javascript
// server.js (line ~380)
const token = jwt.sign({ username: user.username, role: role }, config.jwt.secret, { expiresIn: config.jwt.expiresIn });

// auth.js (line ~25)
const parts = token.split(':');
if (parts.length !== 3) {
  return res.status(401).json({
    success: false,
    error: '无效的令牌格式'
  });
}
```

**影响范围**: 
- 即使修复了问题 1，登录接口返回的 JWT token 也无法通过 auth 中间件验证
- 需要统一 token 生成和验证逻辑

**修复建议**:
选项 A: 修改 auth 中间件支持 JWT token
选项 B: 修改 login 接口使用 `username:timestamp:signature` 格式

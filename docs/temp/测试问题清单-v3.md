## 根因分析

### nav-control.js 问题分析

代码位置: `/TG/tgservice/admin/nav-control.js`

**问题1: 角色字段名不匹配**
- 前端代码使用中文角色名: `'收银'`, `'前厅管理'`, `'教练'`
- 后端JWT token中的role字段可能是英文: `'cashier'`, `'front_admin'`, `'coach'`
- 导致 `roleAllowedPages[role]` 返回 `undefined`，角色被当作管理员处理

**问题2: 收银角色允许页面配置错误**
```javascript
'收银': ['index.html', 'cashier-dashboard.html', 'products.html', 'orders.html']
```
- 包含 `index.html`（数据概览），但预期收银不应看到数据概览
- 包含 `orders.html`（订单管理），但预期收银不应看到订单管理

**问题3: 前端缓存问题**
- 浏览器可能缓存了旧版本的 `nav-control.js`
- 需要清除缓存或添加强制刷新机制

### 后端登录拦截分析

**生效部分**: 
- 服务员角色被正确拦截（BUG-020已修复）
- 错误提示: "服务员不允许登录后台管理系统，请使用前台系统"

**未生效部分**:
- 登录成功后，前端菜单过滤未执行
- 直接URL访问可能未被后端拦截（需进一步验证）

## 建议修复方案

### 1. 统一角色字段名
```javascript
// 方案A: 后端返回中文角色名
// 方案B: 前端使用英文角色名
const roleAllowedPages = {
  'admin': 'all',
  'store_manager': 'all',
  'coach_manager': 'all',
  'front_admin': ['cashier-dashboard.html', 'products.html', 'vip-rooms.html'],
  'cashier': ['cashier-dashboard.html', 'products.html'],
  'coach': ['water-boards.html']
};
```

### 2. 修复收银角色配置
```javascript
'收银': ['cashier-dashboard.html', 'products.html']  // 移除index.html和orders.html
```

### 3. 添加强制刷新机制
```javascript
// 在HTML中添加缓存控制
<meta http-equiv="Cache-Control" content="no-cache, no-store, must-revalidate">
<script src="nav-control.js?v=20260409"></script>
```

### 4. 后端API权限校验
- 在每个管理API中添加角色校验
- 拒绝非授权角色的API请求

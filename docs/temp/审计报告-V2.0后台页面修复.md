**🔴 严重问题**：

#### BUG-CD-001: 筛选未限制2小时窗口（严重）
- **位置**: `cashier-dashboard.html` — `refreshData()` + `renderAll()`
- **问题**: 需求文档要求「已完成（2小时内）」、「已取消（2小时内）」，但代码在筛选「已完成」和「已取消」时**没有任何时间过滤**，会显示所有历史数据。
- **需求依据**: V2.0 需求文档「标签页切换：待处理，已完成（2小时内），已取消（2小时内）」
- **修复建议**: 在 `renderProductOrders()`、`renderServiceOrders()`、`renderCoachSessions()` 中，当筛选状态为「已完成」或「已取消」时，过滤掉 `created_at` 超过2小时的记录。

**🟡 中等问题**：

#### BUG-CD-002: 警报逻辑不精确（中等）
- **位置**: `cashier-dashboard.html` — `refreshData()` 末尾
- **问题**: 红色警报/声音警报在**有任何待处理订单**时就触发，而不是针对「超时/紧急」订单。需求文档暗示应该有超时警报。
- **当前代码**: `newPending > 0 && alertEnabled` 触发警报
- **修复建议**: 增加超时判断逻辑，例如待处理超过15分钟的订单才触发红色闪烁警报。

#### BUG-CD-003: 收银权限包含商品管理（中等）
- **位置**: `nav-control.js` L85 + `permission.js` PERMISION_MATRIX
- **问题**: 需求文档明确「收银权限：收银看板」，但代码中收银权限包含 `['cashier-dashboard.html', 'products.html']`，收银员可以访问商品管理页面。
- **需求依据**: V2.0 需求文档「1.1 后台用户权限划分 → 收银权限：收银看板」
- **修复建议**: `nav-control.js` 中收银权限改为 `['cashier-dashboard.html']`；`permission.js` 中收银的 `productManagement` 改为 `false`。

## P0 - 严重问题

### BUG-009: Docker 镜像过旧，与主机代码不一致（延续BUG-001）
- **问题描述**: Docker 容器中的 server.js 是 3540 行旧版本，主机代码已是 3762 行新版本。V2.0 新增 API 路由（如 `/api/admin/users/:username/permissions`）在 Docker 环境中不可用。
- **影响范围**: 
  - 生产环境（Docker）缺少 V2.0 新增路由
  - 开发环境（PM2）功能正常
- **修复建议**: 
  ```bash
  docker build -t mameisong/tgservice:latest /TG
  docker restart tgservice
  ```
- **状态**: ❌ 待修复
- **关联**: BUG-001

### BUG-010: 后台用户CRUD操作因数据库锁失败
- **问题描述**: 执行用户创建/更新/删除操作时返回 500 错误。错误日志显示 `SQLITE_ERROR: cannot start a transaction within a transaction`，原因是 service-orders.js 使用了 `beginTransaction()` 但数据库连接模式可能存在问题。
- **影响范围**:
  - `POST /api/admin/users` - 500 错误
  - `PUT /api/admin/users/:username` - 500 错误
  - `DELETE /api/admin/users/:username` - 500 错误
  - `POST /api/service-orders` - 空需求校验时超时（事务锁）
- **临时修复**: 重启 PM2 进程可临时恢复，但后续操作仍可能触发锁
- **状态**: ❌ 待修复

### BUG-011: 新增助教时 water_boards 表未自动初始化
- **问题描述**: 调用 `POST /api/admin/coaches` 创建新助教时，coaches 表正确插入记录，但 water_boards 表没有自动初始化该助教的水牌记录。
- **影响范围**: 新创建的助教没有初始水牌状态
- **复现步骤**:
  1. `POST /api/admin/coaches` 创建新助教
  2. 查询 `water_boards` 表，没有该助教的记录
- **状态**: ❌ 待修复

### BUG-012: TC-007 check-scan-permission 不支持后台用户判断
- **问题描述**: `/api/auth/check-scan-permission` 接口只支持通过 `coach_no` 参数查询助教信息，不支持判断后台用户（店长/收银/教练等）的免扫码权限。
- **影响范围**: 
  - 后台用户（非助教）无法通过此接口判断是否可免扫码下单
  - 不符合 TC-007 用例预期（后台用户应返回 `can_order_without_scan: true`）
- **修复建议**: 扩展接口支持后台用户角色判断，或通过 JWT token 中的 role 字段判断
- **状态**: ❌ 待修复

- **问题描述**: API 参数使用 `employeeId`（驼峰），但数据库字段为 `employee_id`（下划线）。新增助教时参数映射正确，但批量修改班次等功能可能存在字段名不一致问题。
- **影响范围**: 助教管理相关 API 的字段名一致性
- **状态**: ⚠️ 需确认

### BUG-015: 服务单空需求校验导致超时
- **问题描述**: 当提交空需求的服务单时，API 超时未返回（而不是返回 400 错误）。可能是事务锁导致请求挂起。
- **影响范围**: 服务单创建 - 输入校验
- **状态**: ❌ 待修复

### BUG-016: 助教班次批量修改功能未在 server.js 主路由中注册

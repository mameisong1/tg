## 开发建议

### 可以进入开发阶段

设计方案已经过 3 轮审计，所有问题均已修订完毕。设计文档完整、逻辑清晰、细节充分，可以开始开发。

### 开发优先级建议

**Phase 1 - 基础框架（1-2 周）**:
1. 数据库表创建和迁移脚本
2. 后台用户管理和登录认证
3. 基础权限校验中间件
4. OSS 上传配置

**Phase 2 - 核心功能（2-3 周）**:
1. 水牌管理（状态转换、上下桌）
2. 助教管理（班次管理）
3. 服务单和上下桌单
4. 操作日志记录

**Phase 3 - 申请审批（1-2 周）**:
1. 加班申请/审批
2. 公休申请/审批
3. 乐捐报备
4. 约客记录上传/审查

**Phase 4 - 统计报表（1 周）**:
1. 约客统计生成
2. 乐捐一览
3. 操作日志查询

**Phase 5 - 前端优化（1-2 周）**:
1. H5/小程序页面开发
2. 状态按钮交互
3. 实时通知推送

### 开发注意事项

1. **事务处理**: 水牌状态变更和上下桌单创建必须在同一事务中
2. **并发控制**: 水牌状态更新需要加锁（建议使用 SQLite 事务锁）
3. **缓存策略**: 约客统计结果需要缓存，缓存键设计为 `guest_invitation_stats:{date}:{shift}`
4. **时间校验**: 约客记录提交需要严格校验截止时间
5. **权限校验**: 所有 API 必须在中间件层校验用户权限
6. **日志记录**: 所有状态变更必须记录操作日志

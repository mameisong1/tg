## 补充建议

### 1. 新增测试用例建议

**权限管理扩展**：
```
TC-025: 管理员角色权限验证
  - 验证管理员可访问全部菜单
  - 验证管理员可进行全部操作

TC-026: 服务员角色权限验证
  - 验证服务员无法访问任何后台菜单
```

**约客管理扩展**：
```
TC-027: 约客审查通过验证
  - 审查结果为"约客有效"
  - 记录审查人和审查时间

TC-028: 约客审查驳回验证
  - 审查结果为"约客无效"
  - 助教可重新上传覆盖

TC-029: 约客审查权限验证
  - 非授权角色无法审查

TC-030: 约客统计生成验证（早班）
  - 16:00 后生成统计
  - 应约客人数正确（首次生成时快照状态）
  - 统计结果缓存

TC-031: 约客统计重复生成验证
  - 重复调用返回缓存结果
  - 不重新快照
```

**申请事项扩展**：
```
TC-032: 早加班申请提交验证
  - 上传加班截图
  - 状态为"待处理"

TC-033: 加班审批通过验证
  - 审批后水牌状态变更为"早加班"
  - 记录操作日志

TC-034: 加班审批拒绝验证
  - 审批拒绝后状态不变

TC-035: 公休申请提交验证
  - 上传公休证明
  - 状态为"待处理"

TC-036: 公休审批通过验证
  - 审批后水牌状态变更为"公休"

TC-037: 公休审批拒绝验证
  - 审批拒绝后状态不变

TC-038: 乐捐报备提交验证
  - 提交后状态自动为"有效"
  - 水牌状态变更为"乐捐"

TC-039: 乐捐报备一览验证
  - 查询最近 10 天乐捐记录
```

**服务单扩展**：
```
TC-040: 服务单创建验证
  - 助教/后台用户可创建服务单
  - 状态为"待处理"

TC-041: 服务单处理验证
  - 状态变更为"已完成"
  - 记录处理时间

TC-042: 服务单取消验证
  - 状态变更为"已取消"
```

**上下桌单扩展**：
```
TC-043: 上桌单提交验证
  - 选择台桌号和类别
  - 水牌状态变更为"早班上桌"/"晚班上桌"

TC-044: 下桌单提交验证
  - 水牌状态变更为"早班空闲"/"晚班空闲"
  - 清空台桌号

TC-045: 取消单提交验证
  - 水牌状态变更为"早班空闲"/"晚班空闲"
  - 清空台桌号

TC-046: 上下桌单处理验证
  - 状态变更为"已完成"
```

**操作日志扩展**：
```
TC-047: 操作日志记录验证
  - 水牌状态变更记录日志
  - 约客审查记录日志
  - 日志包含旧值和新值

TC-048: 操作日志查询验证
  - 按操作人查询
  - 按操作类型查询
  - 按时间范围查询
```

### 2. 异常流程测试建议

```
TC-049: 并发状态变更测试
  - 同时修改同一水牌状态
  - 验证锁机制生效

TC-050: 越权访问测试
  - 收银角色尝试访问水牌管理 API
  - 返回权限不足错误

TC-051: 超时提交测试
  - 16:00:01 提交早班约客
  - 返回精确错误提示

TC-052: 数据校验失败测试
  - 提交空数据
  - 提交非法格式数据
```

### 3. 用例质量改进建议

1. **前置条件标准化**：每个用例应明确数据准备步骤
   - 示例："创建测试账号（角色：店长）"
   - 示例："准备 3 个不同状态的教练水牌"

2. **预期结果具体化**：补充具体错误提示文案
   - 示例：错误提示应为"早班约客上传已截止（16:00）"

3. **测试数据可追溯**：建议使用可识别的测试数据
   - 示例：测试账号统一使用 1380013800X 号段

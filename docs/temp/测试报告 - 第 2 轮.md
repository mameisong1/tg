## 关键发现

### 1. 代码已修复但未部署 ✅🔴

**状态**：
- ✅ 代码层面：server.js 已注册所有 V2.0 路由（第 297-301 行）
- ✅ 路由文件：water-boards.js、coaches.js 等已创建且完整
- 🔴 生产环境：Docker 容器内运行的是旧代码

**验证方法**：
```bash
# 代码已注册
grep -n "water-boards" /TG/tgservice/backend/server.js
# 输出：297:app.use('/api/water-boards', waterBoardsRouter);

# 但 API 不可用
curl http://localhost:8081/api/water-boards
# 输出：Cannot GET /api/water-boards
```

**根本原因**：Docker 容器已运行 11 小时，期间代码修改未重新构建镜像或重启容器。

**解决方案**：
```bash
# 需要执行（但 QA 工程师无权操作）
docker restart tgservice
# 或重新构建镜像
docker build -t mameisong/tgservice:latest /TG
docker restart tgservice
```

### 2. 路由路径变更 ⚠️

**注意**：coaches 路由注册在 `/api/coaches/v2` 而不是 `/api/coaches`。

**影响**：
- 旧 API `/api/coaches` 仍然存在（server.js 第 733 行）
- 新 API 使用 `/api/coaches/v2` 前缀
- 测试时需要注意区分

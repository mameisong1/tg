## 同步脚本改进

### 问题现象

| 现象 | 原因 |
|------|------|
| a. 点击输入过快导致网络异常 | 固定等待时间太短，无随机停顿 |
| b. 页面不停刷新 | 等待时间短、检测次数少、轻易抛异常重试 |

### 解决方案

#### 1. 添加随机停顿函数

```javascript
function randomSleep(minMs = 200, maxMs = 500) {
  const delay = Math.floor(Math.random() * (maxMs - minMs + 1)) + minMs;
  return new Promise(resolve => setTimeout(resolve, delay));
}
```

#### 2. 登录流程改进

| 操作 | 修改前 | 修改后 |
|------|--------|--------|
| 点击密码登录后 | 固定 2000ms | 随机 1500-2500ms |
| 填写用户名后 | 无等待 | 随机 300-600ms |
| 填写密码后 | 固定 1000ms | 随机 500-1000ms |
| 登录检查 | 2 次 | 最多 3 次，每次等 3-5 秒 |

#### 3. 页面检测改进

| 改进项 | 修改前 | 修改后 |
|--------|--------|--------|
| 导航后等待 | 5 秒固定 | 8-12 秒随机 |
| 元素检测次数 | 1-2 次 | 最多 5 次 |
| 检测间隔 | 5 秒固定 | 5-10 秒随机 |
| 超时处理 | 直接放弃 | 多次检查后放弃 |

#### 4. 错误处理增强

```javascript
// 完整错误输出
log('=== 同步失败 ===');
log(`错误类型: ${err.name}`);
log(`错误消息: ${err.message}`);

log(`耗时: ${elapsed}ms`);

// 页面调试信息

```

## 经验教训

1. **台客多页面加载非常慢**，至少要等 10 秒以上才能开始检测元素
2. **不要轻易判定失败**，多次检测后再放弃
3. **操作之间加随机停顿**，防止触发网络异常
4. **错误日志要完整**，包括页面状态、URL、内容预览
5. **Docker 部署后**，宿主机 PM2 命令对容器内服务无效

### 天宫智能开关改造
#### 1.需求
把天宫国际球房里的100多盞灯具全部使用智能开关管理。实现根据开台情况自动关灯， 定时自动开灯， 手动批量开灯等功能。

#### 2.db表建立和数据导入
1. 设备开关表
- 字段：开关ID，开关序号，开关标签， 自动关灯开始时间， 自动关灯结束时间。
- 开关ID+开关序号 是唯一的，可以作为主键
- 注意，这里的开关ID是一个大开关， 里面含了N个小开关，用开关序号来识别小开关
- 自动关灯时间段，代表在这个时间段要执行自动关灯指令（详见下文）。
2. 台桌设备表
- 字段：台桌名英文， 开关序号，开关标签
- 台桌名英文+开关序号+开关标签 是唯一主键
- 说明：说明台桌周边有哪些灯具设备。
3. 开关场景表
- 字段：场景名称（主键）， 动作（开灯/关灯）， 开关数组
- 开关数组是一个json数据，里面有所有常见相关开关的信息， 例如[{"开关ID":"0xa4c1380bbd942cbe","开关序号":"state_l4"}]
4. 数据导入
- 设备开关表的初始数据： /TG/docs/智能开关改造/SwitchManagement.xlsx 文件的【设备开关表】页面里面可以获取到初始数据，请导入；
- 台桌设备表初始数据：/TG/docs/智能开关改造/SwitchManagement.xlsx 文件的【设备开关表】页面，最后一列关联台桌可以获取到与开关设备相关的台桌英文名，一个开关设备和台桌是多对多的关系。请据此生成初始数据导入表里。

#### 3. 后台数据管理
- 后台admin系统目录里面新增3个菜单：
- 设备开关管理：CURD
- 台桌设备关系：CURD
- 开关场景表：CRD， 其中新增场景时， 开关数组数据是用多行文本输入的，每个数组元素一行， 开关ID 和 开关序号之间用TAB键分割。  写入db前自动转换为json格式。

#### 4. 自动关灯功能
##### 4.1 触发时机
后台admin台桌状态同步接口，接口每次被调用时，如果数据正确更新了40条以上台桌数据，立即同步执行自动关灯功能。

##### 4.2 查询确定要关的灯
1. 找出所有：可能要关的灯；
select distinct C.开关ID C.开关序号
from 台桌表 A join 台桌设备表 B join 设备开关表 C 
where A是空闲状态 and 当前时间 between C.自动关灯时间段

2. 查出所有：不能关的灯
select distinct C.开关ID C.开关序号
from 台桌表 A join 台桌设备表 B join 设备开关表 C 
where A不是是空闲状态 and 当前时间 between C.自动关灯时间段

3. 确定要关的灯集合=可能要关的灯集合 -  不能关的灯集合

##### 4.3 对确定要关的所有灯发出关灯指令
系统通过MQTT发出关灯指示(注意测试环境下禁止发出指令，只能写日志)
```

  "id": "$开关ID",
  "$开关序号": "OFF"
}
```

#### 5. 前台H5页面
在前台H5 的管理功能新增智能开关按钮。
1. 智能开关页面功能：
- 【自动关灯功能】 启动/暂停；
- 开关场景表里的每个场景生成一个按钮， 点击后对开关数组，进行action动作（开灯/关灯)；
- 针对 设备开关表 的开关标签， 进行开灯/关灯操作。
下拉框选择开关标签后，对该标签的数个开关进行开灯/关灯操作。
2. 页面设计：要求设计精美，具有科技感，易操作。和H5其它的管理页面风格差异不能过大。。
3. 权限级别：该页面只有店长和助教管理拥有权限。

#### 6. 编码注意事项
1. 所有的时间处理，都要使用现有的时间类；
2. 所有的db操作，都要使用现有的唯一db连接；
3. 所有的db写入操作，都要使用writequeue队列；

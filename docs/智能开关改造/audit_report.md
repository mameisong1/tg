### S2：自动关灯时间段 SQL 判断逻辑错误（跨午夜场景）

| 项目 | 内容 |
|------|------|
| **依据** | README 第4.2节 + design.md 第4.2节 |
| **问题描述** | design.md 中两个 SQL 使用 `TIME(?) >= TIME(sd.auto_off_start) AND TIME(?) <= TIME(sd.auto_off_end)` 判断当前时间是否在自动关灯时间段内。这种简单比较**无法处理跨午夜时间段**（如 22:00~06:00） |
| **举例** | 自动关灯时段 22:00~06:00，当前时间 23:00：`23:00 >= 22:00` ✅ 但 `23:00 <= 06:00` ❌ → 错误判断为不在时段内 |
| **修复方案** | 需判断跨午夜场景，SQL 逻辑应改为：<br>`(sd.auto_off_start <= sd.auto_off_end AND TIME(?) >= TIME(sd.auto_off_start) AND TIME(?) <= TIME(sd.auto_off_end))` <br>OR<br>`(sd.auto_off_start > sd.auto_off_end AND (TIME(?) >= TIME(sd.auto_off_start) OR TIME(?) <= TIME(sd.auto_off_end)))` |

**影响范围**：
- design.md 第4.2节 SQL 需修改
- 自动关灯核心逻辑

### S2：跨午夜SQL ✅ 已修正

| 检查项 | 第一轮问题 | 修正后状态 |
|--------|-----------|-----------|
| SQL逻辑 | 简单 `TIME(?) >= start AND TIME(?) <= end` 无法处理跨午夜 | ✅ 使用 CASE 表达式正确处理 |
| 跨午夜判断 | 无 | ✅ `start > end` 时用 OR，`start <= end` 时用 AND |
| 覆盖范围 | 仅自动关灯 | ✅ 自动关灯（4.2节）+ 定时自动开灯（4A.3节）均使用相同逻辑 |

**验证结果**：design.md 4.2节和4A.3节的SQL均使用了正确的CASE表达式：
```sql
CASE
  WHEN sd.auto_off_start <= sd.auto_off_end THEN
    (TIME(?) >= TIME(sd.auto_off_start) AND TIME(?) <= TIME(sd.auto_off_end))
  ELSE
    (TIME(?) >= TIME(sd.auto_off_start) OR TIME(?) <= TIME(sd.auto_off_end))
END
```
此逻辑可正确处理跨午夜（如 22:00~06:00）和不跨午夜（如 08:00~22:00）两种场景。

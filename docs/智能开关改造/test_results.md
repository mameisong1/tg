## Bug 汇总

### Bug-001：PUT更新API要求所有必填字段

| 项目 | 内容 |
|------|------|
| **标题** | 设备开关PUT更新接口要求传入所有必填字段 |
| **复现步骤** | 1. 创建一条设备开关记录<br>2. PUT /api/admin/switches/:id，只传 `{switch_label: "新标签"}`<br>3. 返回400 `{"error":"缺少必填字段"}` |
| **预期结果** | 更新成功，仅修改传入的字段 |
| **实际结果** | 返回400错误，要求必须传 switch_id、switch_seq、switch_label |
| **影响范围** | TC-004、TC-022 |
| **修复建议** | 修改 switch-routes.js 第79-81行，PUT更新时不应要求所有必填字段，只需验证记录存在即可 |
| **代码位置** | `/TG/tgservice/backend/routes/switch-routes.js` 第77-100行 |

### Bug-002：自动关灯触发阈值与用例预期不一致（需确认）

| 项目 | 内容 |
|------|------|
| **标题** | 自动关灯触发阈值为 >= 40，测试用例TC-042预期为 > 40 |
| **代码行为** | `if (totalUpdated < 40)` → 40条及以上触发 |
| **用例预期** | TC-042：更新恰好40条时应不触发 |
| **建议** | 与需求方确认"40条以上"是否包含40条。如不包含，需改为 `< 41` |
| **代码位置** | `/TG/tgservice/backend/routes/switch-routes.js` 第339行 |

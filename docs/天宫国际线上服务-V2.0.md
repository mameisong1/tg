## 天宫国际v2.0开发

#### DB修改
1. 后台用户表
- 新增姓名字段，权限扩充（管理员/店长/助教管理/教练/前厅管理/收银/服务员）
- 用户名将录入手机号，初始密码随机设置。这些用户一般不登录后台，只用于分配前台的权限。前台用户登录时用手机号来这个表查询权限。
2. 新增 水牌表
- 助教工号，艺名，水牌状态（早班上桌/早班空闲，晚班上桌/晚班空闲，早加班，晚加班，休息，公休，请假， 乐捐，下班），上桌台桌号。
- 用于管理助教的实时工作状态
2. 新增 服务单
- 下单时间，台桌号，需求，下单人（助教艺名，后台用户姓名），状态（待处理，已完成，已取消），生成时间，更新时间。
- 用于助教呼叫服务员来提供服务。
3. 新增 上下桌单
- 下单时间，台桌号，助教工号，单名（上桌单/下桌单/取消单），上桌类别（普通课/标签课），助教艺名，状态（待处理/已完成/已取消），生成时间，更新时间。
- 助教上桌时需要提交上桌单给收银去录入台客多系统处理。下桌时也要提交下桌单。上桌单提交后，还可以提交上桌取消单。
4. 新增申请事项表
- 申请人（手机号），申请事项（早加班申请/晚加班申请/公休申请/乐捐报备/约客记录），备注，申请时间，状态（0待处理/1同意或有效/2拒绝或无效），审批人（手机号），审批时间，生成时间，更新时间。
- 用于管理助教的早加班申请，晚加班申请，公休申请，乐捐报备，约客记录提交。
- 注意，乐捐报备时，不需要审批，状态只有1有效。
5. 助教表
- 新增字段班次（早班/晚班）
- 后台助教管理页面，新增批量修改早晚班状态功能。
6. 约客结果表
- 日期，早班，工号，艺名，约客记录图片url，结果（未约客/约客有效/约客无效），创建时间，更新时间

### 1.员工权限管理
#### 1.1 后台用户权限划分
- 助教表和后台用户表里的用户，统称为员工。
- 其中后台用户表权限分配如下
- 店长，助教管理权限：全部菜单
- 前厅管理的后台权限：收银看板（详见下文），商品管理，包房管理
- 收银权限：收银看板
- 其它后台用户，没有后台权限，不能登录后台。


#### 1.2 前台内部专用版块权限划分
- 我的页面【助教专用】版块，改名为【内部专用】版块
- 登录时检查手机号在后台用户表和助教表的存在。判断登录前台的用户是助教还是店长，助教管理，教练，或者其他后台用户，都显示内部专用版块。版块内容如下。
- 助教可见按钮：个人中心，上班，下班，上桌单，下桌单，上桌取消单，加班申请，公休申请，乐捐报备，上传约客记录。
- 店长和助教管理可见按钮：加班审批（件数），公休审批（件数），乐捐报备一览，早班约客审查（件数），晚班约客审查（件数），早班约客统计， 晚班约客统计，水牌管理。另外，助教管理也显示助教按钮，但是不可操作。
- 教练可见按钮：水牌查看（不可修改）
- 所有后台用户都可见按钮：服务下单。


### 2. 商品下单和服务下单
前台登录（包括自动登录）时，用手机号去后台用户表查询权限。
所有后台用户都有权限进行免扫码下单和服务下单。
#### 2.1免扫码下单
- 未扫码进入时，商品一览，购物车，会员中心不再出现扫码提示。
- 不限制添加商品到购物车，不限制购物车下单功能。
- 下单时如果没有台桌号，弹出台桌输入对话框，要求录入台桌号后再下单。
- 台桌输入对话框要简单易用，先筛选区域（大厅，包房，斯诺克等区域），再用按钮选择台桌。
- 在上桌的助教打开台桌输入框时，默认选中当前上桌的台桌号。
- 以上所有改动都只针对后台用户权限。一般顾客必须先扫码后下单的逻辑不变。

#### 2.2 服务下单功能
- 我的页面点击服务下单按钮，弹出服务下单对话框。
- 对话框组件：桌号输入，需求输入。
- 需求输入预设快捷按钮，换垃圾袋，搞卫生，音响连接等等常用标签按钮。

#### 2.3 收银看板
商品下单，服务下单以及助教的上下桌单，都在收银看板合并显示。由收银员统一调度服务员处理。
- 全屏显示，浏览器去头，页面炫酷，标题：天宫国际 收银看板
- 标签页切换：待处理，已完成（2小时内），已取消（2小时内）。
- 订单卡片，服务单卡片，上下桌卡片，分3列显示。每类1列，显示不下的出现纵向滚动条。类别标题上面显示卡片总数量。
- 订单修改/取消，（需求调研）


### 3.助教管理
#### 3.1 水牌管理
水牌管理页面是用于管理/查看助教实时状态页面。
- 水牌状态定义：早班上桌，早班空闲，晚班上桌，晚班空闲，早加班，晚加班，休息，公休，请假， 乐捐，下班；
- 乐捐定义：是指助教和客人外出状态。
- 水牌页面显示：助教工号+艺名，页面分段显示，每段显示一个状态，从上到下分别显示：
- 页面设计要求炫酷。在前台手机端和后台都做一个水牌页面。
- 助教管理可以手动修改移动每个助教的水牌状态，要求易操作。
- 教练，助教管理，店长都可以看到助教水牌。

影响水牌状态的事件：
- 助教上班，水牌跳到早班空闲或晚班空闲（具体早班还是晚班由助教表.班次决定）。
- 助教下班，水牌跳到下班
- 助教上桌单发送时，水牌跳到早班上桌或晚班上桌（具体早班还是晚班由助教表.班次决定）。
- 助教下桌单或者上桌取消单发送时，水牌跳到早班空闲或晚班空闲（具体早班还是晚班由助教表.班次决定）。
- 公休审批同意时，水牌跳到公休，
- 加班审批同意时，水牌跳到早班加班或晚班加班（具体早班还是晚班由助教表.班次决定）。
- 乐捐报备提交时，水牌跳到乐捐。
- 请假（手动），休息（手动）


#### 3.2 助教新增管理事项
上下班管理
- 上班处理：助教上班时要求点上班按钮，系统根据助教表当前是早班还是晚班，设置水牌为早班空闲或晚班空闲状态。
- 下班处理：助教下班时要求点下班按钮，系统把水牌改为下班状态。
- 可以点上班按钮的水牌状态：早加班，晚加班，休息，公休，请假， 乐捐，下班；（状态不对时按钮变灰）
- 可以点下班按钮的水牌状态：早班上桌，早班空闲，晚班上桌，晚班空闲。（状态不对时按钮变灰）

上下桌管理
- 上桌单提交：助教上桌时要提交上桌单，选择台桌号，选择上桌类型（普通课/标签课)，点击上桌，提交到db.上下桌单表。同时系统更新水牌为早班或晚班上桌状态，并录入上桌台桌号。最后由收银把上桌信息录入到台客多后台系统进行计费。（台桌号录入功能，共用未扫码下单时的页面设计和代码）
- 上桌取消单提交：上桌状态的助教可以取消上桌，提交上桌取消单到db.上下桌单表。同时系统更新水牌为早班或晚班空闲状态，并清空上桌台桌号。
- 下桌单提交：上桌状态的助教，下桌时要点击下桌按钮，系统提交下桌单到db.上下桌单表。同时系统更新水牌为早班或晚班空闲状态，并清空上桌台桌号。
- 注意：已上桌助教不能再提交上桌单。未上桌助教不能取消上桌和下桌。

各种申请
- 助教加班后可以申请晚到（申请时要提交加班截图证明），通宵加班后可以申请一天公休（申请时要提交加班截图证明），系统要提交早晚加班申请/公休申请到【申请事项表】。
- 具体是早加班申请，还是晚加班申请，由助教表.班次来决定。
- 加班截图证明上传到阿里云oss的目录oss.tmpDir （配置文件.config）。由前端直接上传，上传即提交申请成功，不需要额外点保存按钮。 具体代码可参考助教个人中心页面的助教上传功能。
- 助教管理可以审批助教的加班申请，查看加班截图后，点击同意或拒绝。同意后，修改助教水牌为早加班或晚加班（早加班申请变为早加班，晚加班申请变为晚加班）。
- 助教和客人外出要做乐捐报备（输入日期和外出小时数），系统要提交乐捐报备到【申请事项表】。乐捐报备完成后，不需要审批，水牌立刻变成乐捐状态。
- 助教管理可以查阅最近10天的乐捐报备一览（日期，时间，工号，艺名，外出小时数）。

#### 3.3. 助教约客管理
- 要求助教每天约客，并定时传约客截图证明（早班上传时间16:00， 晚班约客证明上传时间20:00)。
- 助教管理在这两个时间点分别审查早晚班助教的约客证明，可以对证明判定有效或无效。
- 审查完毕后，生成早班/晚班的约客统计结果（写入db.约客结果表）：早班（晚班）， a应约客人数，b已约客人数，c无效约客助教一览，d未约客助教一览。
- 应约客助教定义：只有在生成统计结果时点，水牌状态是早班空闲/晚班空闲的助教才需要约客。
- 无效约客和未约客的助教,随时可以补充上传约客证明，助教管理审查后，可以重新生成约客统计。约客统计结果更新时，不更a,只更新bcd；
- 每天只有16:00以后 并且 早班约客记录都审查完毕，才能打开 早班约客统计。
- 每天只有20:00以后 并且 晚班约客记录都审查完毕，才能打开 晚班约客统计。
- 约客截图证明上传到阿里云oss的目录oss.tmpDir （配置文件.config）。由前端直接上传，上传即提交成功，不需要额外点保存按钮。 具体代码可参考助教个人中心页面的助教上传功能。

### 4. V2.0后台页面修复记录（2026-04-09）

#### 4.1 收银看板修复
- **发声提醒优化**：每10秒轮询检查，有待处理订单（商品单/服务单/上下桌单任一）时自动发声提醒收银员（之前仅在新订单到达时发声）
- **工号显示**：上下桌单卡片显示工号(employee_id)而非助教编号(coach_no)，格式为 `工号${employee_id}`
- **侧边栏滚动**：添加 `height:100vh; position:sticky; top:0; overflow-y:auto` 解决菜单栏过长无法滚动的问题
- **测试数据**：当API返回空数据时，自动注入待处理测试数据（商品单2条、服务单3条、上下桌单3条含employee_id）

#### 4.2 批量更新班次修复

- **防重复提交**：使用 `savingShifts` 对象记录正在保存的教练号，防止快速切换导致重复提交
- **布局修复**：修正CSS类名不匹配问题（`coach-avatar`→`batch-shift-avatar` 等）

#### 4.3 菜单名称修改
